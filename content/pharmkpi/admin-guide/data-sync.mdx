# 데이터 동기화

ERP 데이터 수집과 동기화 관리 방법을 설명합니다.

## 동기화 구조

```
사내 ERP (MSSQL) → erp-relay-gateway → PharmKPI API → Supabase (PostgreSQL)
```

PharmKPI는 사내 DB에 직접 접근하지 않습니다. erp-relay-gateway가 MSSQL에서 데이터를 조회하여 JSON으로 전달합니다.

## 동기화 방식

| 방식 | 설명 |
|------|------|
| **자동 (Relay)** | erp-relay-gateway를 통한 API 동기화 |
| **수동 (엑셀)** | 관리자가 엑셀 파일을 직접 업로드 |
| **스냅샷 (Cron)** | `/api/cron/snapshot` 정기 실행 |

## 3단 폴백

| 우선순위 | 방식 | 상태 |
|---------|------|------|
| 1 | VPN 직접 연결 | 미해결 |
| 2 | erp-relay-gateway | 운영 중 |
| 3 | 엑셀 업로드 | 항시 가용 |

## 데이터 불변 원칙

- **과거 월 데이터**: 변경 불가. 캐시에서만 제공
- **당월 데이터**: 수동 동기화로 갱신 가능

## 연결 상태 확인

사이드바에서 **설정 > DB 연결 상태**를 선택하면 다음 항목을 확인할 수 있습니다:

| 항목 | 설명 |
|------|------|
| **Supabase** | DB 연결 상태 |
| **ERP Relay** | erp-relay-gateway 응답 상태 |
| **마지막 동기화** | 가장 최근 데이터 동기화 시각 |

## API 엔드포인트

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/api/health` | GET | 서비스 상태 확인 |
| `/api/sync` | POST | 수동 동기화 트리거 |
| `/api/cron/snapshot` | GET | 정기 스냅샷 실행 |
| `/api/data/[category]` | GET | 카테고리별 데이터 조회 |
