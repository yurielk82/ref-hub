# EV Motor Reliability

전기차 구동모터/인버터 신뢰성 시험 자동화 플랫폼 `v0.40.0`

## 개요

| 항목 | 내용 |
|------|------|
| **펌웨어** | C (TMS320F2838x DSP, Code Composer Studio) |
| **분석** | Python 3.12 (numpy, scipy, matplotlib, reliability) |
| **하드웨어** | SMC300 모터제어 패키지, PMSM, BLDC |
| **도구** | CCS, UniFlash, JTAG (SDS200i) |

### 웹 대시보드

시뮬레이션 실행, 실시간 모니터링, 신뢰성 분석을 통합 제공하는 웹 기반 대시보드이다. 인증(Google OAuth)  후 5개 메인 라우트와 플로팅 AI 채팅 위젯으로 구성된다.

| 항목 | 내용 |
|------|------|
| **프론트엔드** | Next.js 15 (App Router) + React 19 + TypeScript |
| **UI 프레임워크** | Mantine 7 |
| **API 서버** | FastAPI + Uvicorn (포트 8100) |
| **상태 관리** | Zustand (UI 상태) + TanStack Query v5 (서버 상태) |
| **차트** | Recharts |
| **DB** | Neon PostgreSQL (서버리스, @neondatabase/serverless) |
| **인증** | Neon Auth (Google OAuth, Better Auth) |
| **호스팅** | Vercel (대시보드) + 로컬 에이전트 (시뮬레이션/하드웨어) |

대시보드는 Vercel에 배포되어 브라우저에서 접속하고, 시뮬레이션 실행과 하드웨어 통신은 사용자 PC에서 구동되는 로컬 에이전트(FastAPI)가 담당한다. 브라우저와 에이전트 간 통신은 WebSocket으로 이루어진다.

## 주요 기능

- 모터/인버터 신뢰성 시험 자동화
- Weibull 분석 및 시각화
- 고장 예측 모델 (MTTF, 고장률)
- 시리얼/JTAG 통신 기반 실시간 데이터 수집
- 시뮬레이션 파이프라인 (runner.py, pipeline.py)
- Motor x Control x Mode 3축 조합 통합 시뮬레이션 (단일/배치/스윕)
- Monaco Editor 기반 펌웨어/스크립트 코드 뷰어 + Git 연동
- FMEA (Failure Mode and Effects Analysis) 자동 생성 및 RPN 산출
- HTML 종합 보고서 자동 생성 (Weibull + FMEA + 비교 분석 통합)

## 외부 연동

### Claude API (AI 채팅)

모든 페이지 우하단에 플로팅 AI 채팅 위젯을 제공한다. Anthropic Claude API를 통해 시험 결과 분석, 시뮬레이션 파라미터 추천, 기술 문서 질의를 자연어로 지원한다.

| 기능 | 설명 |
|------|------|
| **시뮬레이션 실행** | 채팅으로 파라미터를 지정하고 시뮬레이션 실행 요청 |
| **결과 해석** | Weibull 분석, FMEA 결과에 대한 자연어 질의 응답 |
| **문서 참조** | 프로젝트 문서 기반 기술 질의 |
| **응답 방식** | SSE (Server-Sent Events) 스트리밍 |

사용자가 자신의 API 키를 에이전트에 설정하는 BYOK (Bring Your Own Key) 방식으로 운영된다.

### WebSocket (실시간 모터 데이터 스트리밍)

로컬 에이전트와 브라우저 간 WebSocket 연결로 실시간 모터 데이터를 10Hz 주기로 스트리밍한다. 속도(RPM), 전류(A), 온도(degC), 토크(N-m) 4채널을 동시에 모니터링하며, 각 측정값이 임계치(80% 경고, 100% 위험)를 초과하면 자동 알림과 비상 정지를 수행한다.

### pyserial (시리얼 UART 통신)

SMC300 모터제어 보드와 PC 간 UART/SCI 시리얼 통신으로 실시간 데이터를 수집한다. 24시간 무인 운전 모드에서 자동 로깅하며, 통신 타임아웃 발생 시 고장 이벤트로 기록한다.

### JTAG/SWD (펌웨어 프로그래밍)

SDS200i 절연 JTAG 에뮬레이터(XDS200-ISO)를 통해 TMS320F2838x DSP에 펌웨어를 업로드한다. UniFlash CLI(`dslite.sh`)로 자동 플래싱하며, Code Composer Studio(CCS)에서 실시간 디버깅을 지원한다.

## 배포

Docker + docker-compose로 프론트엔드(SPA)와 API 서버를 단일 컨테이너에 패키징한다.

| 항목 | 내용 |
|------|------|
| **베이스 이미지** | Python 3.12-slim |
| **빌드 방식** | 멀티스테이지 빌드 (Node.js로 SPA 빌드 → Python 이미지에 정적 파일 포함) |
| **컨테이너 구성** | FastAPI가 SPA 정적 파일 서빙 + API 엔드포인트를 단일 프로세스로 제공 |
| **대시보드 (Vercel)** | Next.js App Router 기반, Vercel에 자동 배포 |
| **로컬 에이전트** | 사용자 PC에서 `python agent.py`로 실행, 하드웨어 통신 및 시뮬레이션 담당 |

Vercel에 배포된 대시보드가 브라우저 측 UI를 제공하고, 시뮬레이션과 하드웨어 제어는 사용자 PC의 로컬 에이전트가 수행하는 하이브리드 아키텍처이다.

## 단위 규칙

SI 단위계 사용: `rad/s` (속도), `A` (전류), `V` (전압), `N·m` (토크), `degC` (온도)
