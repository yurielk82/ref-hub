# 내 정보 관리

CSO 정산 포털에 등록된 업체 정보를 확인하고 수정하는 방법을 안내합니다. 업체 정보, 대표자 정보, 담당자 정보, 비밀번호 등을 관리할 수 있습니다.

## 내 정보 화면 진입

우측 상단의 사용자 아이콘을 클릭한 후 **"내 정보 수정"**을 선택합니다. `/profile` 경로의 프로필 화면이 표시됩니다.

## 수정 가능/불가능 항목

프로필 화면에서 확인할 수 있는 항목과 수정 가능 여부는 다음과 같습니다.

| 항목 | 수정 가능 | 비고 |
|------|:---------:|------|
| 사업자번호 | X | 계정의 고유 식별자, 변경 불가 |
| 회원 유형 | X | 일반회원/관리자 구분, 관리자만 변경 가능 |
| 가입일 | X | 회원가입 승인 일자 |
| 업체명 | O | 사업자등록증 상의 회사명 |
| 대표자명 | O | 사업자등록증 상의 대표자 이름 |
| 주소 | O | 도로명 주소 + 상세 주소 |
| 연락처1 | O | 대표 전화번호 |
| 연락처2 | O | 추가 연락처 |
| 이메일 | O | 정산서 알림 수신 이메일 |
| 이메일2 | O | 추가 이메일 |
| 비밀번호 | O | 현재 비밀번호 확인 후 변경 가능 |

> **주의**: 사업자번호는 계정의 고유 식별자이므로 변경할 수 없습니다. 사업자번호가 변경된 경우 새 사업자번호로 신규 가입 후 관리자에게 기존 계정 삭제를 요청하세요. 자세한 내용은 [자주 묻는 질문](/csoweb/user-guide/checklists)을 참조하세요.

## 정보 수정 방법

프로필 화면의 각 섹션(업체 정보, 대표자 정보, 담당자 정보)은 **독립적으로 저장**됩니다. 한 섹션을 수정하고 저장하면 해당 섹션의 변경사항만 반영되며, 다른 섹션에는 영향을 주지 않습니다.

### 수정 절차

1. 수정하려는 섹션에서 항목의 값을 변경합니다.
2. 해당 섹션의 **"변경"** 버튼을 클릭합니다.
3. 변경이 완료되면 화면 하단에 성공 메시지가 표시됩니다.

### 섹션별 수정 예시

**업체 정보 섹션**

업체명을 수정한 경우, 업체 정보 섹션의 **"변경"** 버튼만 클릭하면 업체명만 업데이트됩니다.

**연락처/이메일 섹션**

연락처1이나 이메일을 수정한 경우, 해당 섹션의 **"변경"** 버튼을 클릭합니다. 다른 섹션의 정보는 변경되지 않습니다.

**주소 섹션**

주소를 변경하려면 **"주소 검색"** 버튼을 클릭하여 새 도로명 주소를 선택한 후, 상세 주소를 입력하고 **"변경"** 버튼을 클릭합니다.

### 변경 버튼 비활성화

수정한 내용이 없으면 **"변경"** 버튼이 비활성화(회색) 상태로 표시됩니다. 기존 값에서 변경된 항목이 있을 때만 버튼이 활성화됩니다.

## 비밀번호 변경

보안을 위해 주기적인 비밀번호 변경을 권장합니다.

### 변경 절차

1. 프로필 화면 하단의 **비밀번호 변경** 영역을 찾습니다.
2. **현재 비밀번호**를 입력합니다.
3. **새 비밀번호**를 입력합니다.
4. **새 비밀번호 확인**에 동일한 비밀번호를 다시 입력합니다.
5. **"비밀번호 변경"** 버튼을 클릭합니다.

### 비밀번호 규칙

| 규칙 | 조건 |
|------|------|
| 최소 길이 | 8자 이상 |
| 필수 조합 | 영문 + 숫자 |
| 제한사항 | 현재 비밀번호와 동일 불가 |

## 프로필 저장 후 반영 시점

프로필에서 수정한 정보는 **즉시 시스템에 반영**됩니다. 다만, 정산서 데이터에는 영향을 주지 않는다는 점에 유의하세요.

| 반영 대상 | 반영 시점 | 설명 |
|----------|----------|------|
| 프로필 화면 | 즉시 | 수정한 값이 바로 표시됨 |
| 로그인 세션 | 즉시 | 상단 메뉴의 업체명 등 즉시 갱신 |
| 이메일 알림 | 즉시 | 변경된 이메일로 다음 알림부터 발송 |
| 정산서 데이터 | 향후 업로드분부터 | 기존 업로드 데이터는 변경되지 않음 |

> **주의**: 정산서 데이터는 업로드 시점의 정보를 기준으로 합니다. 프로필에서 업체명이나 연락처를 수정하더라도 이미 업로드된 과거 정산 데이터에는 반영되지 않습니다. 수정된 정보는 이후에 관리자가 새로 업로드하는 데이터부터 반영됩니다.
