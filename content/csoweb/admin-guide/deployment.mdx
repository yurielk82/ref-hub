# 배포

CSO Web 배포 및 환경 설정 가이드입니다.

## 배포 환경

| 환경 | URL | 브랜치 | 방법 |
|------|-----|--------|------|
| Production | Vercel 프로덕션 도메인 | `main` | `git push` 자동 배포 |
| Preview | Vercel 프리뷰 도메인 | PR 브랜치 | PR 생성 시 자동 |
| Local | `http://localhost:3000` | 개발 브랜치 | `npm run dev` |

## 환경 변수

Vercel 대시보드 또는 `.env.local`에서 설정합니다.

| 변수 | 설명 | 위치 |
|------|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 프로젝트 URL | 프론트/서버 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase 공개 키 | 프론트/서버 |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase 서비스 역할 키 | 서버 전용 |
| `JWT_SECRET` | JWT 토큰 서명 키 | 서버 전용 |
| `RESEND_API_KEY` | Resend 이메일 API 키 | 서버 전용 |

## 배포 절차

1. 기능 브랜치에서 개발 완료
2. PR 생성 → Vercel 프리뷰 배포 자동 생성
3. 프리뷰에서 동작 확인
4. `main` 브랜치에 머지 → 프로덕션 자동 배포
5. Vercel 대시보드에서 배포 상태 확인

## 로컬 개발

```bash
git clone <repo-url>
cd csoweb
npm install
cp .env.example .env.local  # 환경변수 설정
npm run dev
```

## 빌드 확인

```bash
npm run build    # 프로덕션 빌드
npm test         # 전체 테스트 (vitest)
npm run test:e2e # E2E 테스트 (Playwright)
```
