# 포장/출고 관리

셀러의 라이브 방송 후 상품 포장부터 배송 완료까지 출고 작업을 단계별로 관리하는 방법을 안내합니다.

## 출고 상태 흐름

출고 작업은 아래 순서로 진행됩니다.

```
대기 → 포장 중 → 발송 준비 → 발송됨 → 완료
```

| 상태 | 의미 |
|------|------|
| **대기** | 방송이 완료되어 포장 작업 대기 중 |
| **포장 중** | 상품 포장 작업이 진행 중 |
| **발송 준비** | 포장이 완료되어 택배 접수 대기 중 (모바일) |
| **발송됨** | 택배사에 인계하여 운송 중 |
| **완료** | 셀러에게 배송 완료 |

웹과 모바일에서 상태 흐름이 다소 다릅니다. 웹은 상태를 자유롭게 선택할 수 있으며, 모바일은 현재 상태에서 다음 단계로만 순차 진행합니다.

## 출고 목록 조회 — 웹

1. 사이드바에서 **출고 관리**를 클릭합니다
2. 출고 작업 목록이 테이블 형태로 표시됩니다
3. 상단 상태 버튼(전체 / 대기 / 포장중 / 발송 / 완료)으로 필터링합니다

테이블 컬럼: 예약 번호, 예약자, 등록일, 상태, 택배사, 송장번호, 작업

## 출고 상태 변경 — 웹

1. 변경할 작업 행의 **상태 변경** 버튼을 클릭합니다
2. **출고 상태 업데이트** 다이얼로그가 열립니다
3. 예약 번호와 예약자가 상단에 표시되어 작업 대상을 확인합니다
4. **새 상태** 드롭다운에서 변경할 상태를 선택합니다
5. 상태를 **발송됨** 또는 **완료**로 변경하는 경우:
   - **택배사** 필드에 택배사 이름을 입력합니다 (예: CJ대한통운, 롯데택배, 한진택배)
   - **송장번호** 필드에 송장번호를 입력합니다
   - 두 항목 모두 필수입니다. 미입력 시 저장이 불가합니다
6. **저장** 버튼을 클릭합니다

발송 전 단계(대기 → 포장중)는 택배사·송장번호 입력 없이 바로 저장할 수 있습니다.

## 출고 목록 조회 — 모바일

1. 하단 탭에서 **운영** > **출고**로 이동합니다
2. 상단 칩(전체 / 대기 / 포장 중 / 발송 준비 / 발송됨)으로 상태를 필터링합니다

각 카드에는 셀러 이름, 예약번호, 현재 상태, 택배사·운송장번호(있는 경우)가 표시됩니다.

## 출고 상태 변경 — 모바일

모바일은 현재 상태에서 다음 단계로만 순차 진행합니다.

1. 해당 작업 카드 하단의 **[다음 상태](으)로 변경** 버튼을 탭합니다
   - 예: 대기 상태이면 **포장 중(으)로 변경** 버튼이 표시됩니다
2. **발송됨**으로 변경하는 경우, 발송 정보 입력 카드가 나타납니다
   - **택배사** 입력 (예: CJ대한통운)
   - **운송장번호** 입력
   - **발송 처리** 버튼을 탭합니다
   - 취소하려면 **취소**를 탭합니다
3. 그 외 단계는 버튼 탭 즉시 상태가 변경됩니다

완료 상태에 도달한 작업은 더 이상 상태 변경 버튼이 표시되지 않습니다.

## 출고 처리 기준

| 단계 | 처리 기준 |
|------|----------|
| 대기 → 포장 중 | 방송 종료 후 상품 수거를 시작할 때 |
| 포장 중 → 발송 준비 | 포장이 완료되어 박스 봉인까지 마쳤을 때 (모바일) |
| 발송 준비 → 발송됨 | 택배사에 실물을 인계하고 운송장을 부착했을 때 |
| 발송됨 → 완료 | 셀러에게 배송이 완료되었음을 확인했을 때 |
