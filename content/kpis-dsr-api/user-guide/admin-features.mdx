# 관리자 기능

관리자 역할이 부여된 계정에서만 접근할 수 있는 기능입니다. API 설정, 마스터 DB 관리, 사용자 관리, 데이터 관리, 휴일 관리, 활동 로그를 포함합니다.

> **주의**: 인증이 비활성화된 환경에서는 모든 사용자가 관리자 권한을 갖습니다.

## API 설정

사이드바 도구 그룹의 **API 설정**을 클릭하면 API 설정 다이얼로그가 열립니다. KPIS API 연동에 필요한 인증 정보를 등록하고 관리합니다.

### 공동인증서 탭

KPIS API 등록(MA111~MA114) 시 전자서명에 사용되는 공동인증서를 관리합니다.

| 기능 | 설명 |
|------|------|
| **인증서 업로드** | DER 포맷 인증서 파일을 업로드합니다 |
| **만료일 자동 파싱** | 업로드 시 인증서에서 만료일을 자동으로 추출합니다 |
| **만료 임박 경고** | 만료 30일/7일/1일 전 단계별 경고 배너 표시 |
| **인증서 삭제** | 기존 인증서를 삭제합니다 |

인증서는 서버에 AES-256으로 암호화하여 보관됩니다.

> **주의**: 공동인증서가 만료되면 KPIS API 자동 제출이 불가능합니다. 만료 전에 반드시 새 인증서를 등록하세요. KPIS 전용 공동인증서(용도제한)를 사용해야 하며, 범용 공동인증서는 사용할 수 없습니다.

### DEV API 탭

개발/테스트 환경용 KPIS API 설정입니다.

| 설정 항목 | 설명 |
|-----------|------|
| **사업자등록번호** | 개발용 사업자등록번호 (10자리, 하이픈 없이 숫자만) |
| **API KEY** | KPIS 개발 포털에서 발급받은 API 키 |
| **연결 테스트** | 설정된 정보로 KPIS 개발 서버 연결을 테스트합니다 |

연결 테스트 성공 시 초록색 점이 표시됩니다.

### LIVE API 탭

운영 환경용 KPIS API 설정입니다. DEV API 탭과 동일한 구조이며, 운영용 사업자등록번호와 API KEY를 등록합니다.

> **주의**: LIVE 환경의 API KEY로 제출하면 실제 KPIS에 보고됩니다. 테스트 데이터를 LIVE 환경에 제출하지 않도록 주의하세요.

### DEV/LIVE 환경 전환

사이드바 하단의 **DEV/LIVE 배지**를 클릭하여 환경을 전환할 수 있습니다.

| 환경 | 대상 서버 | 용도 |
|------|----------|------|
| **DEV** | `devopenapi.kpis.or.kr` | 개발/테스트 — 제출해도 실제 반영되지 않음 |
| **LIVE** | `openapi.kpis.or.kr` | 운영 — 제출하면 실제 KPIS에 보고됨 |

LIVE 모드로 전환 시 경고 다이얼로그가 표시되며, 사이드바 상단에 빨간색 **"LIVE 운영 서버"** 경고 스트라이프가 표시됩니다.

> **주의**: DEV 키로 LIVE 서버를 호출하거나 그 반대는 인증 오류가 발생합니다. 환경 전환 시 반드시 해당 환경의 API KEY가 등록되어 있는지 확인하세요.

## 마스터 DB 관리

사이드바 도구 그룹의 **마스터 DB**를 클릭하면 마스터 데이터 관리 다이얼로그가 열립니다. 표준코드 매칭, 규격 검증, 코드검색에 사용되는 기준 데이터를 관리합니다.

### 표준코드 마스터

KPIS 13자리 의약품 식별코드 데이터입니다. 두 가지 방법으로 등록할 수 있습니다.

**방법 1: KPIS API 조회 (MA101)**

1. 마스터 DB 패널에서 **API 조회** 버튼을 클릭합니다.
2. KPIS MA101 API를 호출하여 최신 표준코드 데이터를 가져옵니다.
3. 기존 데이터와 병합됩니다.

**방법 2: 엑셀 업로드**

1. 마스터 DB 패널에서 **엑셀 업로드** 버튼을 클릭합니다.
2. 표준코드마스터 엑셀 파일을 선택합니다.
3. 파일의 데이터가 마스터 DB에 등록됩니다.

### 바코드 데이터

BarCodeData 엑셀을 업로드하여 규격, 업체명, 제형 등 추가 정보를 등록합니다. 바코드 데이터는 자동 코드매핑의 필터 체인에서 제형구분, 포장수량 등의 비교에 활용됩니다.

### 검색

등록된 마스터 데이터를 제품명 또는 표준코드로 검색할 수 있습니다. 검색 결과에서 각 제품의 포장수량, 상한가, 규격, 제형 등의 상세 정보를 확인할 수 있습니다.

> **주의**: 마스터 DB가 비어있거나 오래된 데이터만 있으면 자동 코드매핑의 매칭률이 크게 떨어집니다. 분기 보고 시작 전에 최신 데이터로 업데이트하는 것을 권장합니다.

## 사용자 관리

사이드바 도구 그룹의 **사용자관리**를 클릭하면 사용자 관리 다이얼로그가 열립니다. Google OAuth 인증이 활성화된 환경에서만 표시됩니다.

### 사용자 목록

등록된 모든 사용자가 목록으로 표시됩니다. 각 사용자의 이메일, 역할, 승인 상태를 확인할 수 있습니다.

### 관리 기능

| 동작 | 설명 |
|------|------|
| **승인** | 대기 중인 사용자에게 접근 권한을 부여합니다 |
| **승인 취소** | 기존 사용자의 접근 권한을 회수합니다 |
| **삭제** | 사용자 계정을 완전히 삭제합니다 (복구 불가) |
| **역할 변경** | 사용자를 관리자(admin) 또는 일반(user) 역할로 변경합니다 |

> **주의**: 최초 가입자는 자동으로 관리자 역할이 부여됩니다. 관리자가 최소 1명은 유지되어야 합니다.

## 데이터 관리

사이드바 도구 그룹의 **데이터 관리**를 클릭하면 관리 화면(AdminView)으로 이동합니다.

### 날짜별 데이터 집계

날짜별로 업로드된 데이터의 요약 정보가 테이블로 표시됩니다.

| 열 | 설명 |
|----|------|
| **날짜** | YYYY-MM-DD 형식 |
| **갑지** | 갑지 전체 행 수 |
| **을지** | 을지 전체 행 수 |
| **보고** | KPIS에 제출 완료된 건수 |
| **미보고** | 아직 제출하지 않은 건수 |
| **상태** | 보고완료 / 부분보고 / 업로드만 / 오류 배지 표시 |

### 관리 기능

| 동작 | 설명 |
|------|------|
| **행 클릭** | 해당 날짜의 데이터를 DataGrid에 로드합니다 |
| **상태 초기화** | 접수번호, 제출상태, 오류를 모두 초기화합니다. 데이터를 재보고 가능한 상태로 되돌립니다 |
| **삭제** | 해당 날짜의 갑지/을지 데이터를 완전히 삭제합니다 |

> **주의**: **삭제**는 복구할 수 없습니다. 이미 KPIS에 제출한 건이 포함된 날짜를 삭제하더라도 KPIS 서버의 보고 내역은 유지됩니다. KPIS 보고를 취소하려면 삭제가 아닌 취소(코드 5) 제출이 필요합니다.

## 휴일 관리

사이드바 도구 그룹의 **휴일 관리**를 클릭하면 휴일 관리 다이얼로그가 열립니다.

### 기능

| 기능 | 설명 |
|------|------|
| **공휴일 자동 수집** | 공공데이터포털 API를 통해 법정 공휴일 데이터를 자동으로 가져옵니다 |
| **회사 휴일 설정** | 회사 자체 휴무일을 수동으로 추가합니다 |
| **휴일 삭제** | 등록된 휴일을 삭제합니다 |

### 활용

- 달력의 날짜 색상 점에 영업일/휴일 구분이 반영됩니다.
- 미보고 날짜 배너에서 휴일은 자동으로 제외됩니다.
- 보고율 계산에서 휴일은 제외됩니다.

## 활동 로그

상단 헤더의 **활동 로그** 버튼(벨 아이콘)을 클릭하면 활동 로그 다이얼로그가 열립니다.

### 기록되는 활동

시스템의 주요 활동이 자동으로 기록됩니다.

| 활동 유형 | 예시 |
|-----------|------|
| 데이터 업로드 | "2026-03-08 갑지 150건 업로드" |
| 자동처리 | "코드매핑 45건, 규칙 12건 적용" |
| API 제출 | "MA111 출고 120건 제출 완료" |
| 결과확인 | "MA112 118건 등록, 2건 반송" |
| 데이터 삭제 | "2026-03-08 데이터 삭제" |
| 마스터 업데이트 | "표준코드마스터 1,885건 등록" |
| 환경 전환 | "API 환경 DEV → LIVE 전환" |

활동 로그는 시간순으로 정렬되며, 각 항목에 발생 시각과 상세 내용이 표시됩니다.
