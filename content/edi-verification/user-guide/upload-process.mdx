# 업로드 및 처리

ZIP 파일 업로드부터 OCR 처리 완료까지의 전체 과정을 설명합니다.

## Step 1: 파일 업로드

### ZIP 파일 준비

CSO별 폴더로 구성된 ZIP 파일을 준비합니다:

```
2026-03_처방통계.zip
├── A업체/
│   ├── 서울병원_202603.jpg
│   └── 부산약국_202603.pdf
├── B업체/
│   ├── 대구병원_202603.png
│   └── 인천약국_202603.jpg
└── C업체/
    └── 광주병원_202603.pdf
```

> **주의**: 폴더명이 CSO 업체명과 일치해야 자동 매칭됩니다. 정확한 명칭 사용을 권장합니다.

### 지원 파일 형식

| 형식 | 확장자 | 비고 |
|------|--------|------|
| 이미지 | .jpg, .jpeg, .png | 단일 페이지 |
| PDF | .pdf | 다중 페이지 지원 (페이지별 이미지 분리) |

### 업로드 절차

1. 드롭존에 ZIP 파일을 드래그앤드롭 (또는 클릭하여 선택)
2. ZIP 내부 구조가 트리뷰로 표시됩니다
3. **처방월**을 선택합니다 (예: 2026-03)
4. **다음** 버튼으로 Step 2로 이동

### 중복 파일 처리

동일 배치 내에서 이미 업로드된 파일(Perceptual Hash 기준)은 자동으로 건너뛰며, 건너뛴 파일 목록이 표시됩니다.

## Step 2: CSO 매칭

ZIP 폴더명을 기반으로 CSO 업체를 자동 매칭합니다.

### 자동 매칭

- 폴더명과 등록된 CSO 업체명의 유사도를 계산합니다 (pg_trgm)
- 유사도가 높은 업체를 자동 선택합니다
- 매칭 결과를 확인하고 필요 시 드롭다운으로 수정합니다

### 수동 매칭

자동 매칭에 실패한 폴더는 드롭다운에서 직접 CSO 업체를 선택합니다. 모든 폴더가 매칭되어야 처리를 시작할 수 있습니다.

### 처리 시작

- **모든 CSO 매칭 완료** 상태에서 **처리 시작** 버튼이 활성화됩니다
- 클릭하면 OCR 파이프라인이 시작됩니다

## Step 3: 처리 진행

### 실시간 진행률

SSE(Server-Sent Events)로 실시간 처리 상황을 표시합니다:

| 표시 항목 | 설명 |
|-----------|------|
| 진행률 바 | 전체 파일 중 처리 완료 비율 |
| 자동확인 | OCR 신뢰도 임계값 이상으로 자동 통과된 건 |
| 플래그 | 검수가 필요한 건 (신뢰도 미달 또는 위변조 의심) |
| 오류 | 처리 실패 건 |

### 일시정지/재개

처리 중 **일시정지** 버튼으로 파이프라인을 중단할 수 있습니다. 재개 시 중단된 지점부터 계속합니다.

### 오류 건 재처리

오류가 발생한 파일은 개별 **재시도** 버튼으로 재처리할 수 있습니다.

### 처리 완료

모든 파일 처리 완료 시:
- 배치 상태가 `review_pending`으로 변경됩니다
- 플래그된 건이 있으면 **검수하기** 링크가 표시됩니다
- 엑셀 리포트 다운로드가 가능합니다

## 처리 이력

하단 **처리 이력** 섹션에서 과거 배치를 확인할 수 있습니다:

| 기능 | 설명 |
|------|------|
| 배치 목록 | 처방월, 상태, 파일 수, 처리 일시 |
| 엑셀 다운로드 | 배치별 검증 결과 엑셀 |
| 삭제 | 배치 및 관련 데이터 전체 삭제 |
