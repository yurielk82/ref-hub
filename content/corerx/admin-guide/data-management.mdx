# 데이터 관리

데이터 관리 탭과 시스템 관리 탭의 운영 방법을 설명합니다. UBIST 데이터 업로드, 캐시 상태 확인, 사용자 권한 관리, 매출 목표 설정을 다룹니다.

## 데이터 관리 탭

데이터 관리 탭에서는 캐시 상태와 회사명 매핑 현황을 확인할 수 있습니다. 이 탭은 영업관리자 역할에게만 표시됩니다.

### 캐시 상태 확인

캐시 상태 테이블은 각 캐시 테이블의 마지막 갱신 시각, 상태, 행 수를 표시합니다.

| 컬럼 | 설명 |
|------|------|
| 캐시 테이블 | 캐시 테이블명 (cache_sales_monthly 등) |
| 마지막 갱신 | 마지막 갱신 완료 일시 |
| 상태 | completed / running / failed |
| 행 수 | 캐시 테이블의 현재 행 수 |
| 트리거 | 갱신을 유발한 이벤트 (ubist_upload / manual 등) |

### 상태별 의미

| 상태 | 의미 | 조치 |
|------|------|------|
| completed | 정상 갱신 완료 | 정상 상태 |
| running | 갱신 진행 중 | 대기 (대용량 데이터 시 수 분 소요 가능) |
| failed | 갱신 실패 | [문제 해결](/corerx/admin-guide/troubleshooting) 참조 |

### 회사명 매핑 현황

UBIST 데이터의 제조사명과 자사 시스템의 회사명이 일치하는지 확인합니다. 매핑이 누락되면 시장점유율 계산에서 자사 데이터가 빠질 수 있습니다.

## 데이터 업로드

### UBIST 데이터 업로드

UBIST 처방조제 데이터는 대용량 엑셀 파일(약 96MB)이므로, CLI(Command Line Interface)를 통해 업로드합니다.

| 항목 | 내용 |
|------|------|
| 파일 형식 | UBIST_D1 Sales.xlsx (Excel) |
| 파일 크기 | 약 96MB |
| 행 수 | 약 110,000행 |
| 컬럼 수 | 191개 (10개 차원 + 3개 지표 x 60개월) |
| 업로드 방식 | CLI 스크립트 (`scripts/ubist_import.py`) |
| 갱신 주기 | 분기 1회 (UBIST 데이터 입수 시) |

상세한 ETL 실행 방법은 [ETL 파이프라인](/corerx/admin-guide/etl-pipeline) 문서를 참조하세요.

### 데이터 삭제

특정 연도의 UBIST 데이터를 삭제해야 하는 경우 CLI에서 실행합니다.

```bash
python3 scripts/ubist_import.py --delete-year 2021
```

> **주의**: 데이터 삭제 후 반드시 캐시를 갱신해야 합니다. 삭제만 하고 캐시를 갱신하지 않으면 대시보드에 이전 데이터가 표시됩니다.

## 시스템 관리 탭

시스템 관리 탭에서는 사용자 승인, 역할 관리, 매출 목표 설정을 할 수 있습니다. 영업관리자 역할에게만 편집 권한이 있습니다.

### 사용자 승인

Google OAuth로 최초 로그인한 사용자는 승인 대기 상태가 됩니다.

1. 시스템 관리 탭의 **사용자 목록** 테이블에서 대기 중인 사용자를 확인합니다.
2. 사용자의 소속과 역할을 확인한 후 적절한 역할을 부여합니다.
3. 승인이 완료되면 해당 사용자에게 이메일 알림이 발송됩니다.

### 역할 관리

사용자에게 부여할 수 있는 역할은 다음과 같습니다:

| 역할 | 설명 | 데이터 범위 |
|------|------|-------------|
| 경영진 | 전사 실적 조회 | 전체 데이터 읽기 |
| 영업관리자 | 전체 관리 권한 | 전체 데이터 읽기/쓰기 |
| 영업사원 | 본인 영업소 실적 | 소속 부서 데이터만 |
| CSO | 본인 CSO 성과 | 소속 CSO 데이터만 |

### 매출 목표 설정

연간 매출 목표를 설정하여 달성률을 모니터링할 수 있습니다.

1. 시스템 관리 탭의 **매출 목표** 테이블에서 현재 설정된 목표를 확인합니다.
2. 제품별, 기간별 목표 금액을 입력합니다.
3. 목표가 설정되면 경영 요약 탭과 제품 성과 탭에서 달성률(AC/PL)이 자동 계산됩니다.

> **주의**: 매출 목표를 변경하면 관련된 모든 달성률 지표에 즉시 반영됩니다. 연도 중 목표 변경 시 이력 관리에 유의하세요.

## 데이터 갱신 주기 요약

| 데이터 | 갱신 주기 | 방법 | 담당 |
|--------|----------|------|------|
| ERP 주문서 | 월 1회 | `import_erp_orders.py` 실행 | 운영자 |
| 갑지 공급내역 | 실시간 동기화 | KPIS 사이트 자동 동기화 | 자동 |
| UBIST 처방 | 분기 1회 | `ubist_import.py` 실행 | 운영자 |
| CSO 정산 | 월 1회 | CSO Web 사이트 자동 동기화 | 자동 |
| 캐시 테이블 | 데이터 변경 시 | `refresh_all_kpi_cache()` 자동/수동 | 자동/운영자 |
