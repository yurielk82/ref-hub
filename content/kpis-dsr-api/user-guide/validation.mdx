# 검증

시스템은 데이터를 편집할 때마다 실시간으로 유효성 검증을 수행합니다. 오류가 있는 행은 KPIS API 제출이 차단되므로, 제출 전에 모든 오류를 해결해야 합니다.

## 3단계 검증 구조

검증은 세 단계로 순차적으로 수행됩니다.

| 단계 | 검증 유형 | 설명 |
|------|----------|------|
| **1단계** | 필수값 검증 | 빈 값이 있으면 안 되는 필수 필드를 확인합니다 |
| **2단계** | 형식 검증 | 값이 존재하더라도 자릿수, 범위, 포맷이 올바른지 확인합니다 |
| **3단계** | 비즈니스 검증 | 마스터 DB 기준 표준코드 존재 여부, 수량 정합성 등 업무 규칙을 확인합니다 |

### 1단계: 필수값 검증

누락되면 안 되는 기본 필드를 확인합니다.

| 검증 항목 | 규칙 |
|-----------|------|
| 연번 (A열) | 빈 값 불가 |
| 공급사업자등록번호 (B열) | 빈 값 불가 |
| 공급구분 (E열) | 빈 값 불가 |
| 공급받은자 명칭 (G열) | 빈 값 불가 |
| 제품명 (J열) | 빈 값 불가 |
| 표준코드 (K열) | 빈 값 불가, 13자리 존재 필수 |
| 공급수량 (M열) | 빈 값 불가, 0보다 커야 함 |
| 공급일자 (N열) | 빈 값 불가 |
| 공급금액 (O열) | 빈 값 불가, 숫자여야 함 |
| 공급단가 (P열) | 빈 값 불가, 0 이상 |
| 반송사유 (R열) | 반품(2)/폐기(3) 시 필수 |

### 2단계: 형식 검증

값의 자릿수, 범위, 포맷을 확인합니다.

| 검증 항목 | 형식 규칙 |
|-----------|----------|
| 표준코드 (K열) | 정확히 13자리 숫자 |
| 사업자등록번호 (B열, H열) | 10자리 형식 |
| 공급일자 (N열) | YYYYMMDD 8자리 |
| 유효기간 (T열) | YYMMDD 6자리 |
| 요양기관기호 (I열) | 공급형태=5일 때 8자리 필수 |
| 영업형태 (C열) | 1~7 범위 |
| 계약방법 (D열) | 1 또는 2 |
| 공급형태 (F열) | 1~9 범위 |
| 일련번호구분 (U열) | 0, 1, 2 중 택일 |

### 3단계: 비즈니스 검증

데이터 간의 정합성과 KPIS 기준 준수 여부를 확인합니다.

| 검증 항목 | 규칙 |
|-----------|------|
| 표준코드 존재 여부 | 마스터 DB에 등록된 코드인지 확인 |
| 단가 x 수량 정합성 | 단가 × 수량 ≈ 금액 (1원 이내 오차 허용) |
| 폐기 금액 제로 | 공급구분=3(폐기)일 때 단가와 금액이 반드시 0원 |
| 을지 필수 여부 | 일련번호구분=0인데 을지 데이터가 없으면 오류 |
| 요양기관 기관기호 | 공급형태=5일 때 요양기관기호 8자리 필수 |

## 오류 코드 분류

시스템 내부 검증에서 발생하는 주요 오류 코드는 다음과 같습니다.

### DC 계열: 코드 관련 오류

| 코드 | 오류명 | 원인 | 해결 방법 |
|------|--------|------|----------|
| **DC001** | 표준코드 불일치 | 13자리 표준코드가 마스터 DB에 없거나 만료됨 | 코드검색 다이얼로그에서 정확한 표준코드 확인 후 수정 |
| **DC002** | 미등록 거래처 | KPIS에 등록되지 않은 사업자등록번호 | 거래처 사업자번호 확인 후 수정 |
| **DC003** | 보고기간 외 공급일자 | 해당 분기 보고기간에 포함되지 않는 날짜 | 공급일자를 보고기간 내 날짜로 수정 |
| **DC004** | 중복보고 | 동일 데이터가 이미 KPIS에 등록됨 | 정정(코드 4) 또는 취소(코드 5)로 처리 |
| **DC005** | 수량/금액 오류 | 단가 × 수량 ≠ 금액 정합성 불일치 | 단가, 수량, 금액 재확인 후 수정 |
| **DC006** | 을지 누락 | 일련번호구분=0인데 매칭되는 을지 데이터 없음 | 을지 파일 재업로드 또는 일련번호구분을 1로 변경 |
| **DC007** | 요양기관기호 오류 | 공급형태=5(요양기관)인데 기관기호 누락 또는 8자리 아님 | 정확한 8자리 요양기관기호 입력 |

### DE 계열: 수량/금액 관련 오류

| 코드 | 오류명 | 원인 | 해결 방법 |
|------|--------|------|----------|
| **DE001** | 수량 이상 | 공급수량이 0 이하이거나 비정상적으로 큰 값 | 정확한 수량 입력 |
| **DE002** | 단가 이상 | 공급단가가 음수이거나 비정상적으로 큰 값 | 정확한 단가 입력 |
| **DE003** | 금액 정합성 | 단가 × 수량과 금액의 차이가 허용 오차(1원) 초과 | 규격보정 실행 또는 수량/단가 수동 조정 |

### DH 계열: 기타 오류

| 코드 | 오류명 | 원인 | 해결 방법 |
|------|--------|------|----------|
| **DH001** | 필수 필드 누락 | 필수 입력 항목이 비어있음 | 해당 필드에 값 입력 |
| **DH002** | 형식 오류 | 자릿수, 범위, 포맷이 규칙에 맞지 않음 | 올바른 형식으로 재입력 |
| **DH003** | 비즈니스 규칙 위반 | 공급구분별 특수 규칙 위반 (폐기 시 금액 ≠ 0 등) | 해당 공급구분 규칙에 맞게 수정 |

## 오류와 경고의 차이

| 구분 | 의미 | 표시 | 제출 가능 여부 |
|------|------|------|---------------|
| **오류** | 필수값 누락, 형식 불일치, 비즈니스 규칙 위반 | 빨간색 좌측 테두리 | 제출 불가 — 반드시 수정 필요 |
| **경고** | 잠재적 문제 (단가 이상, 사업자번호 형식 권장, 제조번호 길이) | 노란색 좌측 테두리 | 제출 가능 — 확인 권장 |

주요 경고 항목은 다음과 같습니다.

| 경고 항목 | 규칙 |
|-----------|------|
| 사업자등록번호 형식 | 10자리 형식 권장 (오류는 아님) |
| 금액 정합성 | 단가 × 수량과 금액의 1원 이상 차이 |
| 반품/폐기/취소 반송사유 | 반송사유 기재 권장 |
| 제조번호 길이 | 20자리 이하 권장 |

## 오류 확인 방법

### 검증 바 (DataGrid 하단)

DataGrid 하단에 검증 요약 바가 표시됩니다. 오류 건수와 경고 건수가 각각 표시되며, 바를 클릭하면 상세 목록이 펼쳐집니다.

1. 검증 바를 클릭하여 상세 목록을 펼칩니다.
2. **오류** 탭과 **경고** 탭으로 구분되어 표시됩니다.
3. 각 항목의 연번을 클릭하면 해당 행으로 스크롤됩니다.
4. 오류 내용과 해결 방법이 함께 표시됩니다.

### 행 직접 확인

빨간색 좌측 테두리가 표시된 행을 클릭하면 검증 다이얼로그가 열리며, 해당 행의 모든 오류와 경고를 상세히 확인할 수 있습니다.

## 오류 일괄 해결

### 자동처리 버튼

상단 헤더의 자동처리 관련 버튼으로 여러 오류를 한 번에 해결할 수 있습니다.

| 버튼 | 기능 | 해결하는 오류 |
|------|------|-------------|
| **규격검증** (자 아이콘) | 마스터 DB 기준 규격/수량/단가 자동 보정 | DE003 (금액 정합성), 규격 불일치 |
| **코드매핑** | 미매칭 표준코드 자동 채움 | DC001 (표준코드 불일치) |
| **규칙관리** | 배치 규칙 일괄 적용 | 조건에 맞는 다양한 오류 |

### 규격검증 상세

**규격검증** 버튼(자 아이콘)을 클릭하면 마스터 DB의 표준규격과 현재 공급규격을 비교하여 불일치 건을 자동 보정합니다.

- 불일치 건이 있으면 버튼이 **빨간색**으로 표시됩니다.
- 불일치 건이 없으면 버튼이 **초록색**으로 표시됩니다.
- 보정 시 공급금액(총액)은 불변 유지하면서 규격, 수량, 단가를 재계산합니다.
