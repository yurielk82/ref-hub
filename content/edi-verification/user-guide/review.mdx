# 검수

플래그된 건에 대한 사람 검수(Human-in-the-loop) 절차를 설명합니다.

## 검수 진입

검수가 필요한 배치가 있을 때:
- Step 3 완료 화면에서 **검수하기** 링크 클릭
- 또는 처리 이력에서 `review_pending` 상태 배치의 **검수** 버튼 클릭

## 검수 대기열

검수 화면에 진입하면 해당 배치의 플래그된 요청 목록이 표시됩니다.

### 요약 카드

| 항목 | 설명 |
|------|------|
| 전체 | 배치 내 총 요청 수 |
| 검수 대기 | 아직 판정하지 않은 건 |
| OCR 확인 | OCR 결과가 정확하여 확인한 건 |
| 수정 | OCR 오류를 수정한 건 |
| 의심 | 위변조 의심으로 표시한 건 |

### 대기열 필터

- **전체**: 모든 요청
- **검수 대기**: 판정 필요한 건만
- **플래그 사유별**: 신뢰도 미달, ELA 의심, EXIF 의심, 해시 중복

## 검수 상세 화면

요청을 클릭하면 상세 검수 화면으로 이동합니다.

### 레이아웃 구성

| 영역 | 내용 |
|------|------|
| 좌측 | 원본 이미지 뷰어 (확대/축소/회전 지원) |
| 우측 상단 | 요청 정보 (CSO명, 병원명, 파일명, 파이프라인 상태) |
| 우측 중앙 | OCR 추출 항목 테이블 |
| 우측 하단 | 판정 패널 |

### 위변조 지표

상세 헤더에 위변조 분석 결과가 배지로 표시됩니다:

| 지표 | 정상 | 의심 |
|------|------|------|
| ELA 점수 | 임계값 미만 (녹색) | 임계값 이상 (적색) |
| EXIF SW | 감지 없음 | Photoshop 등 편집 도구 감지 |
| 해시 중복 | 없음 | 동일 해시 파일 존재 |

### OCR 항목 테이블

추출된 품목 데이터가 테이블로 표시됩니다:

| 컬럼 | 설명 |
|------|------|
| 제조사 | 의약품 제조사명 |
| 청구코드 | 건강보험 청구코드 |
| 품목명 | 의약품명 |
| 단위 | 단위 (정, 캡슐 등) |
| 처방건수 | 해당 기간 처방 건수 |
| 단가 | 단위당 가격 |
| 총수량 | 총 처방 수량 |
| 총금액 | 총 금액 |
| 자사품목 | 자사 품목 여부 (자동 매칭) |
| CSO계약 | CSO 계약 품목 여부 |

### 인라인 편집

OCR 결과에 오류가 있으면 셀을 클릭하여 직접 수정할 수 있습니다:

1. 수정할 셀을 클릭
2. 값을 수정
3. Enter로 확정 (또는 Esc로 취소)
4. 수정된 셀은 하이라이트 표시되며 원본 값이 보존됩니다

> **주의**: 수정 이력은 엑셀 리포트 시트6에 자동 기록됩니다.

## 판정

### 판정 유형

| 판정 | 의미 | 사용 시점 |
|------|------|-----------|
| **OCR 확인** | OCR 결과가 원본과 일치 | 이미지와 추출 데이터가 정확히 일치할 때 |
| **수정** | OCR 오류를 수정하여 확인 | 인라인 편집으로 오류를 바로잡은 후 |
| **의심** | 위변조 의심 표시 | ELA/EXIF/해시 이상 또는 육안으로 변조 의심 시 |

### 판정 절차

1. 원본 이미지와 OCR 데이터를 비교
2. 필요 시 인라인 편집으로 수정
3. 코멘트 입력 (선택사항)
4. 판정 버튼 클릭

### 네비게이션

- **이전/다음** 버튼으로 대기열 내 이동
- 키보드 단축키 지원 (← / →)
- 판정 완료 시 자동으로 다음 건으로 이동

## 배치 확정

모든 플래그 건의 검수가 완료되면:

1. 배치 상태가 `confirmed`로 변경됩니다
2. 최종 엑셀 리포트를 다운로드할 수 있습니다
3. 엑셀에는 수정 이력 시트가 포함됩니다
