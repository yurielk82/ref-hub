# CSO 매칭

CSO 매칭은 정산 엑셀에 기재된 업체명과 포털에 가입된 회원의 사업자번호를 연결하는 작업입니다. 매칭이 완료되어야 정산 조회 화면에서 자사의 정산 데이터를 확인할 수 있습니다.

## CSO 매칭이 필요한 이유

SIT(영업정보시스템)에서 추출한 정산 엑셀에는 **업체명**(텍스트)이 기재되어 있습니다. 반면 포털에 가입한 회원은 **사업자번호**로 식별됩니다. 이 두 가지 정보는 형태가 다르기 때문에, "이 정산 데이터는 이 회원의 것"이라는 관계를 만들려면 업체명과 사업자번호를 연결하는 작업이 필요합니다.

예를 들어, 정산 엑셀에 "ABC영업"이라는 업체명이 있고 포털에 사업자번호 "123-45-67890"으로 가입한 회원이 있을 때, CSO 매칭은 "ABC영업 = 123-45-67890"이라는 연결을 생성합니다. 이 연결이 있어야 해당 회원이 로그인했을 때 "ABC영업"의 정산 데이터를 조회할 수 있습니다.

> CSO 매칭은 관리자(영업관리팀)가 수행하는 작업입니다. 일반회원은 매칭을 직접 수행할 수 없으며, 매칭 상태만 확인할 수 있습니다.

## 매칭 상태 4종류

각 CSO 업체의 매칭 상태는 다음 네 가지 중 하나입니다. 매칭 상태에 따라 정산 데이터 조회 가능 여부가 결정됩니다.

### 정상 (normal)

엑셀의 업체명과 회원의 사업자번호가 정상적으로 연결된 상태입니다. 정산 조회 화면에서 자사 데이터를 확인할 수 있습니다. 별도 조치가 필요하지 않습니다.

### 미등록 (unregistered)

정산 엑셀에 업체명이 존재하지만, 해당 업체가 포털에 회원으로 가입하지 않은 상태입니다. 정산 데이터가 존재하지만 볼 수 있는 회원이 없는 상태입니다.

**대응 방법**:
1. [회원가입](/csoweb/user-guide/getting-started#회원가입)을 진행합니다.
2. 관리자 승인을 기다립니다.
3. 승인 후 관리자가 CSO 매칭을 수행합니다.

### 가입대기 (pending_join)

회원가입 신청서를 제출했으나 관리자 승인이 아직 완료되지 않은 상태입니다. 승인이 완료되면 관리자가 CSO 매칭을 진행합니다.

**대응 방법**:
- 승인까지 영업일 기준 1~2일이 소요됩니다.
- 급한 경우 관리자(영업관리팀)에게 직접 문의하세요.

### 매칭없음 (missing_match)

포털에 회원으로 등록되어 있으나, 정산 엑셀의 업체명과 연결이 되지 않은 상태입니다. 회원 계정은 존재하지만 정산 데이터를 조회할 수 없습니다.

**주요 발생 원인**:
- 신규 가입 후 아직 관리자가 매칭 작업을 진행하지 않은 경우
- 정산 엑셀에 기재된 업체명이 포털에 등록된 회사명과 다른 경우 (예: "ABC" vs "ABC영업")
- 해당 정산월에 업체의 정산 데이터가 아직 업로드되지 않은 경우

**대응 방법**:
관리자에게 아래 정보를 전달하여 매칭을 요청하세요.

| 전달 항목 | 예시 |
|----------|------|
| 사업자번호 | 123-45-67890 |
| 포털 등록 회사명 | ABC영업 |
| 정산 엑셀 상의 업체명 | ABC (엑셀에 기재된 이름과 다를 수 있음) |
| 조회하려는 정산월 | 2026-01 |

## 자신의 매칭 상태 확인 방법

일반회원이 자신의 CSO 매칭 상태를 직접 확인하는 방법은 다음과 같습니다.

### 1단계: 정산 데이터 조회 시도

1. [정산 조회](/csoweb/user-guide/dashboard) 화면에 접속합니다.
2. 정산월 드롭다운에서 조회할 월을 선택합니다.
3. 데이터가 표시되면 매칭이 **정상** 상태입니다.

### 2단계: 데이터가 표시되지 않을 때

정산 조회 화면에 데이터가 나타나지 않는다면 아래 순서로 확인하세요.

1. **정산월 확인**: 다른 정산월을 선택하여 데이터가 있는지 확인합니다. 특정 월에만 데이터가 없을 수 있습니다.
2. **검색 필터 확인**: 검색어가 입력되어 있다면 지워서 전체 데이터를 표시해봅니다.
3. **관리자 문의**: 위 사항을 모두 확인해도 데이터가 없으면 관리자에게 매칭 상태를 문의합니다.

> 관리자가 매칭을 완료하면 정산 조회 화면에 데이터가 즉시 표시됩니다. 별도의 로그아웃/로그인은 필요하지 않습니다.

## 매칭 흐름 요약

회원가입부터 정산 데이터 조회까지의 전체 흐름입니다.

1. CSO 업체가 포털에 **회원가입**을 신청합니다.
2. 관리자가 가입 신청을 **승인**합니다.
3. 관리자가 SIT 엑셀을 포털에 **업로드**합니다.
4. 시스템이 엑셀의 업체명과 기존 매칭 테이블을 **자동 대조**합니다.
5. 이미 매칭된 업체는 자동으로 연결되고, 새 업체는 관리자가 **수동 매칭**합니다.
6. 매칭이 완료되면 해당 회원이 정산 조회 화면에서 **자사 데이터를 조회**할 수 있습니다.

## 매칭과 무결성 검사

관리자는 CSO 매칭의 정합성을 주기적으로 검사합니다. 무결성 검사에서는 다음 항목을 확인합니다.

- 엑셀에 존재하지만 매칭되지 않은 업체 목록
- 매칭은 되어 있으나 해당 정산월에 데이터가 없는 회원 목록
- 동일 업체가 여러 사업자번호에 중복 매칭된 경우

일반회원은 무결성 검사를 직접 수행할 수 없습니다. 매칭 관련 문제가 있으면 관리자(영업관리팀)에게 문의하세요.
