# 문제 해결

운영 중 발생할 수 있는 문제와 해결 방법을 안내합니다.

## 데이터가 표시되지 않음

| 원인 | 확인 방법 | 해결 |
|------|-----------|------|
| DB 연결 끊김 | **설정 > DB 연결 상태** 확인 | Supabase Docker 컨테이너 상태 확인 |
| 동기화 미실행 | 마지막 동기화 시각 확인 | `/api/sync` 수동 트리거 |
| 당월 데이터 미업로드 | 업로드 이력 확인 | 엑셀 업로드 실행 |

## ERP Relay 연결 실패

```bash
# erp-relay-gateway 상태 확인
pm2 status

# 로그 확인
pm2 logs erp-relay-gateway
```

relay 서버가 중단된 경우 PM2에서 재시작합니다. 사내 VPN이 끊어진 경우 엑셀 업로드로 우회합니다.

## 로그인 불가

| 증상 | 해결 |
|------|------|
| "승인 대기" 표시 | 관리자가 **사용자 관리**에서 승인 |
| 비밀번호 분실 | Supabase Auth 비밀번호 재설정 |
| Supabase Auth 오류 | Docker 컨테이너 `supabase-auth` 로그 확인 |

## 업로드 실패

| 원인 | 해결 |
|------|------|
| 양식 불일치 | **템플릿 다운로드**에서 올바른 양식 확인 |
| 파일 크기 초과 | 데이터를 분할하여 업로드 |
| 중복 데이터 | 동일 기간 데이터 기존 업로드 이력 확인 |

## Supabase 컨테이너 문제

```bash
# 전체 컨테이너 상태 확인
docker ps --filter "name=supabase"

# DB 컨테이너 재시작
cd /home/ubuntu/supabase/docker && docker compose restart db

# 전체 재시작
cd /home/ubuntu/supabase/docker && docker compose down && docker compose up -d
```

## AI 기능 오류

| 증상 | 해결 |
|------|------|
| 429 오류 (요청 제한) | 잠시 후 재시도 — API 요청 한도 초과 |
| API 키 오류 | **설정 > AI**에서 키 재입력 |
| 응답 없음 | API 제공사 상태 확인 (Claude/Gemini) |
